## Architecture

**Amber** is a link-saving app (like Pocket) with optional Firebase sync. UI strings are in Italian.

### State Management
Uses `provider` (ChangeNotifier pattern). Two providers:
- `AuthProvider` — wraps Firebase Auth, exposes `user`, `isLoggedIn`, auth methods
- `LinkProvider` — manages the links list, delegates to `LinkRepository`

### Repository Pattern (key abstraction)
`LinkRepository` routes storage calls based on auth state:
- **Logged in** → `FirebaseStorageService` (Firestore: `/users/{uid}/links/{linkId}`)
- **Logged out** → `LocalStorageService` (SQLite: `amber.db`)

On login, `migrateLocalToCloud()` moves all local SQLite links to Firestore and clears the local DB.

### Data Model
`LinkItem` has `id` (UUID v4), `url`, `title`, `createdAt`. The `fromMap()` factory handles two date formats (ISO 8601 string and millisecond timestamp) for backward compatibility.

### Share Intent
The app registers as a share target. `main.dart` sets up a global navigator key and a `receive_sharing_intent` listener that extracts URLs via regex and navigates to `AddLinkScreen` with a pre-filled URL.

### Firebase
Credentials are in `lib/firebase_options.dart` (generated by `flutterfire configure` — replace with your own project). Firestore security rules need to be deployed separately — see `SETUP.md` for production rules.
